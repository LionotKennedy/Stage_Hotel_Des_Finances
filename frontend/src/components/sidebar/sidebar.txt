Je comprends que vous voulez masquer certains éléments du menu de navigation en fonction du rôle de l'utilisateur. Voici comment vous pouvez modifier votre code pour implémenter cette logique :

### Modification du composant Sidebar

Modifiez le composant SideBar comme suit :

```jsx
import React from 'react';
import { Link, useLocation } from 'react-router-dom';
import "./sidebar.scss";
import sidebar_items from '../../Data/Sidebar__route.json';
import logo from '../../assets/images/logo.png';

const SidebarItem = props => {
  const active = props.active ? 'active' : '';
  
  return (
    <div className="sidebar__item">
      <div className={`sidebar__item-inner ${active}`}>
        <i className={props.icon}></i>
        <span>
          {props.title}
        </span>
      </div>
    </div>
  )
}

const SideBar = () => {
  const location = useLocation();

  // Fonction pour filtrer les éléments du menu basé sur le rôle
  const filteredItems = sidebar_items.filter(item => {
    // Si le role est 0, masquer les éléments avec route "/journal" et "/utilisateur"
    if (localStorage.getItem('userRole') === '0') {
      return item.route !== '/journal' && item.route !== '/utilisateur';
    }
    return true; // Sinon, afficher tous les éléments
  });

  const activeItem = filteredItems.findIndex(item => 
    location.pathname.startsWith(item.route)
  );
  
  return (
    <div className='sidebar'>
      <div className="sidebar__logo">
        <img className='logo_img' src={logo} alt="company logo" />
        {/* Bro-Coding */}
      </div>
      {filteredItems.map((item, index) => (
        <Link to={item.route} key={index}>
          <SidebarItem
            title={item.display_name}
            icon={item.icon}
            active={index === activeItem}
          />
        </Link>
      ))}
    </div>
  )
}

export default SideBar;
```

### Explications des modifications

1. Nous avons ajouté une fonction `filteredItems` qui filtre les éléments du menu basé sur le rôle de l'utilisateur.
2. Si le rôle est '0', nous masquons les éléments avec route "/journal" et "/utilisateur".
3. La fonction `filter` est appliquée à `sidebar_items` avant d'afficher les éléments dans le composant.
4. Le reste du composant reste inchangé, mais maintenant il utilise `filteredItems` au lieu de `sidebar_items`.

### Points clés à considérer

1. Assurez-vous que le champ 'userRole' est toujours disponible dans le localStorage après la connexion.
2. Cette approche suppose que vous avez déjà implémenté un système pour stocker le rôle de l'utilisateur dans le localStorage.
3. Vous pouvez facilement ajuster les routes à cacher ou à afficher en modifiant la condition dans la fonction `filteredItems`.

### Exemple d'utilisation

Cette modification permettra de masquer les menus "Journal" et "Utilisateur" si le rôle de l'utilisateur est '0'. Pour d'autres rôles, tous les menus seront affichés normalement.
