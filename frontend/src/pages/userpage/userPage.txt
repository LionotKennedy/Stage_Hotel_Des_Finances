Je comprends maintenant le problème. Vous avez raison, le problème vient du fait que l'URL de l'image contient le chemin complet, y compris le domaine local. Voici une solution simple pour résoudre ce problème :

### Solution proposée

Modifiez la partie `useEffect` comme suit :

```jsx
useEffect(() => {
    if (userData && userData.data) {
        setFormData(prevData => ({
            ...prevData,
            name: userData.data.name || '',
            email: userData.data.email || '',
            image: userData.data.image ? new File([userData.data.image], 'profile.jpg', {type: 'image/jpeg'}) : null
        }));

        // Supprimez le chemin complet et gardez uniquement le nom de l'image
        setImagePreview(userData.data.image ? URL.createObjectURL(new Blob([userData.data.image])) : '');
    }
}, [userData]);
```

Et modifiez la partie `handleSubmit` comme suit :

```jsx
const handleSubmit = async (e) => {
    e.preventDefault();

    let updateData = {};

    if (formData.name !== userData.data.name) {
        updateData.name = formData.name;
    }

    if (formData.email !== userData.data.email) {
        updateData.email = formData.email;
    }

    if (formData.image || !userData.data.image) {
        updateData.image = formData.image ? formData.image : null;
    }

    console.log('FormData:', updateData);

    try {
        await updateUser({ userId, data: updateData }, {
            onSuccess: () => {
                setFormData(prevData => ({
                    ...prevData,
                    name: '',
                    email: '',
                    image: null
                }));
                setImagePreview('');
                navigate('/profile');
                enqueueSnackbar('Utilisateur mis à jour avec succès', {
                    variant: 'success',
                    anchorOrigin: {
                        vertical: 'top',
                        horizontal: 'center',
                    },
                    autoHideDuration: 5000,
                    action: (
                        <IconButton size="small" onClick={() => { }}>
                            <AiOutlineClose fontSize="small" />
                        </IconButton>
                    ),
                    style: {
                        backgroundColor: '#4caf50',
                        color: '#ffffff',
                    },
                });
            },
            onError: (error) => {
                console.error('Erreur lors de la mise à jour de l\'utilisateur:', error.message);
                enqueueSnackbar('Erreur lors de la mise à jour du profil', {
                    variant: 'error',
                    anchorOrigin: {
                        vertical: 'top',
                        horizontal: 'center',
                    },
                    autoHideDuration: 5000,
                });
            }
        });
    } catch (error) {
        console.error('Erreur lors de la mise à jour de l\'utilisateur:', error);
        enqueueSnackbar('Erreur lors de la mise à jour du profil', {
            variant: 'error',
            anchorOrigin: {
                vertical: 'top',
                horizontal: 'center',
            },
            autoHideDuration: 5000,
        });
    }
};
```

### Explications

1. Dans `useEffect`, nous créons toujours un nouveau fichier blob pour l'image existante si elle est présente dans les données utilisateur.

2. Nous utilisons `URL.createObjectURL(new Blob([userData.data.image]))` pour créer une nouvelle URL de prévisualisation qui ne contient que le nom de l'image et pas le chemin complet.

3. Dans `handleSubmit`, nous conservons la même logique pour gérer les mises à jour de l'image.

Ces modifications devraient résoudre le problème de perte de l'image lorsqu'elle n'est pas modifiée et garantir que seule la photo modifiée est enregistrée si l'utilisateur ne change pas son nom ou son adresse e-mail.

Cette approche conserve l'URL de prévisualisation sans le chemin complet, ce qui devrait résoudre le problème de "votre image devient vide" lorsque vous cliquez sur le bouton "handleSubmit".
